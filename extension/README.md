# LingTeX Tools — Browser Extension

Converts FLEx, Phonology Assistant, and custom TSV data to LaTeX as you paste —
directly in Overleaf or any web-based text editor.

## How it works

| Method | How to use |
|--------|-----------|
| **Smart-paste shortcut** | Press **Ctrl+Shift+V** (Windows/Linux) or **⌘⇧V** (Mac) to read the clipboard, convert, and insert at the cursor — always available |
| **Auto-convert paste** | Toggle "Auto-convert paste" in the popup ON, then regular **Ctrl+V** intercepts and converts automatically |

The popup doubles as a configuration and test UI — paste sample data to verify your template before using it in your document.

## Profiles

- **FLEx Interlinear** — converts `\ref`/`\tx`/`\mb`/`\ge` blocks to `\gll…\glt` (langsci-gb4e / gb4e)
- **Phonology Assistant** — TSV rows → LaTeX via configurable row template (`$WORD`, `$GLOSS`, `$ID`, `$COLn`)
- **Dekereke** — same TSV logic, different default column mapping
- **Custom tabs** — create as many as you need via the `+` button

Settings and profiles are stored in `chrome.storage.local` and persist across browser sessions.

## Build / install (development)

### Step 1 — assemble the extension

```bash
cd extension
./build.sh
```

This copies `shared/` files and `docs/core.js` (the shared library) into `chrome/` and `firefox/`.

### Step 2 — load unpacked

**Chrome / Chromium / Edge:**
1. Navigate to `chrome://extensions`
2. Enable **Developer mode**
3. Click **Load unpacked** → select the `extension/chrome/` folder

**Firefox:**
1. Navigate to `about:debugging#/runtime/this-firefox`
2. Click **Load Temporary Add-on** → select `extension/firefox/manifest.json`

For permanent Firefox installation, sign via [addons.mozilla.org](https://addons.mozilla.org/developers/).

## Safari

Convert the Chrome extension to a Safari extension using Apple's tool:

```bash
xcrun safari-web-extension-converter extension/chrome \
    --project-location extension/safari \
    --app-name "LingTeX Tools"
```

Then open `extension/safari/LingTeX Tools/LingTeX Tools.xcodeproj` in Xcode,
build, and enable in Safari → Preferences → Extensions.

Requires macOS 12+, Xcode 14+, and an Apple Developer account for distribution.

## Release zips (GitHub Actions)

The release workflow automatically runs `./build.sh --zip` and attaches
`lingtex-tools-chrome.zip` and `lingtex-tools-firefox.zip` to each GitHub release.

## File structure

```
extension/
  shared/           ← source of truth for all logic
    core.js         ← copied from docs/core.js at build time
    content.js      ← paste intercept + smart-paste shortcut
    background.js   ← routes keyboard command to content script
    popup.html      ← popup UI (same design as web app)
    popup.js        ← popup logic (chrome.storage instead of localStorage)
    icons/          ← add icon16.png, icon48.png, icon128.png here
  chrome/
    manifest.json   ← Manifest V3
    (built files copied here by build.sh)
  firefox/
    manifest.json   ← Manifest V2 (broader compatibility)
    (built files copied here by build.sh)
  safari/           ← generated by xcrun safari-web-extension-converter
  build.sh          ← build script
```
